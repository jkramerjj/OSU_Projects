How to implement Twitter4J OAUTH / POST / GET

If your like me you searched the internet for hours looking for a way to implement OAUTH, POST, and GET for a Twitter application within the Google App Engine / Android Studio environments, but the internet kept sending down rabbit holes. Well you're in the right place now, because this tutorial is designed to walk you through the steps of creating a successful Twitter OAUTH, POST, and GET for your Google App Engine or Android Studio Project.

We are going to start with how to get our OAUTH access strings from Twitter and then we will dive into the Twitter4J Java library.  The purpose here is to show you a quick and effective implementation of OAUTH, GET, and POST via the Twitter4J library. This library is used for both Google App Engine and Android Studio. I will focus some additional tips for Android Studio developers.

In the end you will have functioning Twitter OAUTH/GET/POST code that can be implemented in any Google App Engine or Android device. So get comfortable, prepare to read, prepare to practice, prepare to learn, and, if you've coded at all, prepare to debug.

OK Lets begin:
First we're going to request the 4 required OAUTH credentials from Twitter's website. They are:

  1. Consumer Key (API KEY)
  2. Consumer Secret (API SECRET)
  3. Access Token
  4. Access Token Secret

NOTE: Make sure to copy all 4 OAUTH credentials exactly, because they're kind of important.
